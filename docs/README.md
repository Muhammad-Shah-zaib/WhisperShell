# WhisperShell Documentation

Welcome to the internal documentation for WhisperShell. 
To maintain a clean and scalable project as we expand, our documentation is structured into the following sections:

- **`logger/`**: Contains chronological development logs (e.g., `v1.md`, `v2.md`). These track our progress, bug fixes, and major architectural decisions over time. *(Note: This folder is meant for personal use and is ignored by version control).*
- **`architecture/`**: (Future) Will contain detailed system designs, state machine diagrams, component interactions, and Wayland-specific quirks we document over time.
- **`frontend/`**: (Future) Will contain UI/UX design rules, Tauri frontend event handling specs, styling tokens, and notes on transparent webview implementation.
- **`backend/`**: (Future) Will contain Rust engine specifics, Vulkan backend configuration details, Audio pipeline specifics (`cpal`/`rubato`), and model inference parameters.

For the project design blueprint and motivation, please refer to the `DESIGN.md` in the root of the repository.
