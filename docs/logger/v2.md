# WhisperShell Update Log - v2 Development Session

**Date:** July 13, 2026

This document lists the updates, UI improvements, and critical bug fixes implemented during the development session to streamline the user experience, fix Linux-specific rendering quirks, and prepare the app for metadata branding.

---

## 1. Model Updates & Cleanups
* **Removed Parakeet v3 Model:** Removed the Parakeet v3 model completely from the backend (`src/lib.rs`) and frontend (`ui/components/ModelSelection.js`).
* **Defaulted to Whisper Base:** Configured `whisper-base` (`ggml-base.en.bin`) as the default model when booting the application or generating default configuration files.
* **Simplified Choices:** Restricted the model selection pool to only keep **Whisper Base** (`base`) and **Whisper Turbo** (`turbo`). `Whisper Large v3` and all other testing models were temporarily removed from the client options until further notice.

## 2. Window Close & Toggle Behavior (Crucial Bug)
* **The Issue:** Closing the settings/config window via the native window titlebar cross (`X`) button destroyed the window instance. This caused the background process to remain active but left the GUI inaccessible. Toggling via CLI (`--toggle-config`) or clicking the app icon from the system launcher could no longer reopen the settings because `get_webview_window("main")` returned `None`.
* **The Solution:** Added a Tauri window event listener (`on_window_event`) targeting `CloseRequested` events. Instead of destroying the window, the event is intercepted (`api.prevent_close()`), and the window is safely hidden (`window.hide()`). This preserves the instance, allowing the IPC daemon listener or app launcher to reliably restore the window visibility when toggled.

## 3. UI Overlay Ghosting & Smearing Fixes (Linux WebKitGTK)
* **The Issue:** With `WEBKIT_DISABLE_COMPOSITING_MODE=1` enabled on Linux to bypass driver crashing bugs, the transparent overlay window did not properly clear the background buffer between frame redraws. As a result, when the overlay transitioned states (e.g., from Processing dots to Done checkmark), the old wider state and bouncing dot frames were left smeared/ghosted in the background.
* **The Solution:**
  1. Changed the `.pill` container's background from a semi-transparent `rgba(26, 26, 26, 0.9)` with backdrop filters to a fully solid background `#1a1a1a`. This forces the compositor to overwrite all background pixels inside the container boundaries.
  2. Set a fixed width of `84px` (previously `fit-content`). This ensures that the solid background covers the exact same area during all states (Recording, Processing, and Done), cleanly overdrawing and erasing the previous frames.

## 4. Overlay Bottom Shadow Clipping
* **The Issue:** The overlay pill used `align-items: flex-end` inside its transparent parent container, pushing the pill flush against the bottom edge of the 80px-tall window. Consequently, the bottom shadow (`box-shadow`) was clipped at the window boundaries, rendering a flat, unpolished line.
* **The Solution:** Changed the container alignment to `align-items: center`. This centers the 36px-tall pill vertically within the 80px window, giving the box shadow 22px of padding on both the top and bottom to render completely and smoothly on all sides. We also adjusted the box shadow to a modern, tighter size (`0 2px 10px rgba(0, 0, 0, 0.2)`) to mitigate the Linux alpha-accumulation bug during long animations.

## 5. Website Branding & Package Metadata
* **RPM & Package Upgrades:** Set the homepage URL to the new domain `https://whispershell.tech` in both `Cargo.toml` and `package.json`.
* **Vendor Info:** Updated the author/publisher from "you" to "WhisperShell". When you rebuild the RPM package (`npm run tauri build`), these changes are bundled directly into the installation package metadata so your branding is visible during system double-clicks.
